# wapl-web-service

> 와플은 개발자들의 사이드 프로젝트 팀메이트 매칭을 위한 웹서비스입니다

- 새 프로젝트를 생성하고 팀원을 모집할 수 있습니다
- 주 포지션, 부 포지션을 설정하고 각 수준을 설정할 수 있다.
- 라운지에서는 실시간 댓글 형식으로 대화할수 있다
- 스토리에서는 각자의 경험을 공유할 수 있다
- 와플인에서는 가입자들의 프로필을 보고 프로젝트 제안을 할 수 있다.

### 소감 : 

프론트엔드와 백엔드개발을 동시에 해보니 우선순위 설정에 조금 어려움을 겪었습니다.   
프론트엔드 개발에 치중하다가 백엔드 개발에 조금 소홀했던 것 같습니다.   
정작 중요한 기능을 구현 못했던 부분도 있었습니다.  
이번 프로젝트를 할 때에는 우선순위 설정보다는 눈에 보이는 부분, 매끄럽지 못한 부분, 페이지 이동할때 css변경, 데이터 유지 등에 신경 쓰느라 시간을 많이 잡아먹었는데,  
다음 프로젝트를 진행 할때는 핵심 기능 부터 구현하고 자잘한 부분을 보완하는 식으로 우선순위 설정에 힘을 기울여야 할 것 같습니다.

### 개선점

#### 기능별

1. 즉각적인 노출
   - 토이 프로젝트만을 위한 간단하고 직관적인 UI
     - 플젝제안/구인 게시판
     - 댓글
     - 조인요청/심사&승인
     - 프로필 등록/조회
2. 실시간 피드백
   - 알림기능
     - 관심분야 알림 기능
     - 댓글 알림기능
     - 조인요청 알림기능
3. 프로필 노출
   - 수준별 프로젝트 & 필터링에 도움
     - 프로필 등록
     - 프로필 조회
     - 1대1 채팅
     - 프로젝트 제안

## 입력폼 



- #### 입력폼은 로그인, 회원가입, 프로젝트 생성&수정, 스토리생성&수정 등이 있습니다.

  - 로그인 
    - id가 틀렸는지 password가 틀렸는지 피드백 기능
    - history.back()기능

![image](https://user-images.githubusercontent.com/53857239/129557642-5e75872e-ac06-459d-8dc5-19d3d702252a.png)

- #### 회원가입

  - 아이디는 커서를 떼면 비동기처리로 중복을 체크하고 가입버튼을 비활성화합니다
  - 모두 적당한 정도의 maxlength를 지정해주었습니다. 
  - 프로젝트 참여에 자유도를 위해 주요 포지션, 서브포지션을  지정할 수 있습니다.
  - 업로드한 이미지와 프로필정보는 아래처럼 프로필에 노출됩니다
